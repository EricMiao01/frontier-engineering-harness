# Domain Context

## Harness

The environment around a coding model that supplies context, tools, skills, constraints, state, and verification.

## Skill

A compact, reusable intervention that changes agent behavior for a specific failure mode.

## Workflow

A composition of skills and stages. Workflows are optional and should be selected according to task risk.

## Mechanical enforcement

Executable constraints such as tests, type checks, linters, hooks, architecture checks, and CI.

## Ceremony tax

The extra cost, latency, context consumption, or human burden introduced by a workflow without sufficient quality gain.

## Accepted task

A task whose implementation passes deterministic checks and satisfies human or reference-based review criteria.
